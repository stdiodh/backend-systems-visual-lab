# Content UI Rules

## 핵심 규칙

Visual Lab 화면은 상세 이론 저장소가 아니다. 화면은 문제를 보고, 지표를 읽고, 선택지를 비교하는 학습 도구다.

## 문장 규칙

- 제목은 짧게 쓴다.
- 카드 본문은 2~3줄을 넘기지 않는다.
- 긴 정의는 개념 문서로 보내고 화면에는 요약만 둔다.
- 같은 정보를 여러 카드에 반복하지 않는다.
- 모르는 용어가 나오면 Concept Card 또는 Glossary로 연결한다.

## 장별 구성 규칙

모든 장은 아래 순서를 따른다.

1. 문제 상황
2. 관찰 지표
3. 원인 후보
4. 핵심 개념
5. 해결 선택지
6. 코드/구조 연결
7. 체크리스트

## 금지 사항

- 개념 정의만 나열하는 화면
- 모든 내용을 한 화면에 밀어 넣는 구성
- 긴 본문을 카드에 복사하는 방식
- 문제 상황 없이 바로 용어 설명으로 시작하는 장
- 장점만 있고 부작용이 없는 선택지
- 색상만으로 상태를 전달하는 UI
- 텍스트가 버튼이나 카드 밖으로 넘치는 UI

## 데이터 연결 규칙

- 장 데이터는 `docs/site/chapter-data-spec.md`를 따른다.
- 시나리오 데이터는 `docs/site/scenario-data-spec.md`를 따른다.
- 개념 데이터는 `docs/concepts` 문서의 정의를 기준으로 한다.
- 화면 텍스트가 문서 정의와 충돌하면 문서를 먼저 수정한 뒤 구현한다.
