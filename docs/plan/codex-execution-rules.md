# Codex Execution Rules

## 가장 중요한 규칙

Codex는 매 작업 시작 시 `AGENTS.md`를 가장 먼저 읽는다.

막히거나 모르는 내용이 있으면 추측으로 채우지 말고 사용자에게 질문한다. 특히 커리큘럼 범위, 장별 의도, A&I 중앙 레포와의 관계, 배포 방식은 임의로 결정하지 않는다.

## 작업 전 확인 순서

1. `AGENTS.md`
2. `README.md`
3. `docs/plan/project-plan.md`
4. `docs/plan/phase-roadmap.md`
5. `docs/plan/codex-execution-rules.md`
6. 현재 Phase와 관련된 세부 문서

## Phase 1 실행 규칙

Phase 1은 문서 기반 구축이다.

허용:

- Markdown 문서 작성
- 디렉터리 구조 생성
- 다음 Phase 프롬프트 작성
- 누락 범위 검수

금지:

- HTML 작성
- CSS 작성
- JavaScript 작성
- `site/` 구현 생성
- UI 스크린샷 또는 더미 페이지 생성
- 외부 패키지 설치

## 구현 Phase 실행 규칙

사이트 구현은 Phase 2 이후에만 한다.

구현 시 지킬 것:

- 사이트는 문서 스펙을 기준으로 만든다.
- 한 장에 필요한 데이터는 먼저 구조화한 뒤 렌더링한다.
- HTML에 긴 이론 본문을 직접 넣지 않는다.
- 모든 장은 문제 상황으로 시작한다.
- 모든 선택지는 장점, 부작용, 도입 조건을 함께 보여준다.
- 디자인은 `docs/design` 기준을 따른다.

## 누락 방지 규칙

장별 구현 전 확인:

- `docs/curriculum/chapter-scope.md`의 장 범위를 읽었는가
- 해당 장의 `docs/concepts/*.md` 개념을 읽었는가
- 문제 상황, 관찰 지표, 원인 후보, 해결 선택지, 코드/구조 연결이 모두 있는가

구현 후 확인:

- 커리큘럼 맵의 장 목록이 누락되지 않았는가
- 개념 사전의 필수 개념이 화면에 반영됐는가
- 텍스트가 카드 안에서 넘치거나 겹치지 않는가
- 색상만으로 상태를 구분하지 않는가

## 질문해야 하는 상황

- 사용자가 지정한 범위와 문서가 충돌할 때
- 새 개념을 추가해야 할지 애매할 때
- 중앙 레포에 변경이 필요한지 애매할 때
- 배포 방식이나 레포 이름 결정이 필요한 때
- 외부 도구, 라이브러리, 프레임워크 도입이 필요해 보일 때
