# Project Plan

## 문서 목적

이 문서는 A&I Backend Systems Visual Lab의 전체 목표, 역할, 범위, 산출물 기준을 고정한다.

이 랩은 기존 시퀀스별 Visual Lab이 아니다. 단일 주제의 보조 페이지가 아니라, 실무 백엔드 개발자가 서비스를 운영하면서 마주치는 문제를 하나의 사고 흐름으로 연결하는 통합 학습 랩이다.

## 핵심 콘셉트

학습자는 느려진 서비스를 출발점으로 삼아 지표를 읽고, 병목 후보를 좁히고, 구조적 해결책을 비교한다.

```text
서비스가 느려짐
-> 지표로 병목 후보를 좁힘
-> DB와 쿼리 구조를 확인함
-> 외부 연동 장애 전파를 막음
-> 비동기 처리로 실패 지점을 분리함
-> 동시성으로 인한 데이터 꼬임을 잡음
-> IO 모델을 선택함
-> 보안과 운영 기준을 보강함
-> 서버와 네트워크 기초로 원인을 추적함
-> 구조와 설계 패턴으로 시스템을 정리함
```

## 저장소 책임

이 저장소가 관리하는 것:

- 전체 커리큘럼과 장별 범위
- 실무 개념 사전
- Visual Lab 디자인 시스템
- 사이트 콘텐츠 데이터 구조
- 단계별 Codex 작업 지시서
- 이후 구현될 정적 사이트와 학습 데이터

중앙 레포가 관리하는 것:

- 이 저장소를 부록/심화 자료로 안내하는 짧은 링크 문서
- 기존 정규 시퀀스와 서브모듈 관리

## Phase 1 산출물

Phase 1은 문서 기반 구축만 한다.

생성 대상:

- `README.md`
- `AGENTS.md`
- `docs/plan/*`
- `docs/curriculum/*`
- `docs/concepts/*`
- `docs/design/*`
- `docs/site/*`
- `docs/prompts/*`

생성하지 않는 대상:

- `site/index.html`
- `site/styles/*.css`
- `site/scripts/*.js`
- 구현용 이미지나 더미 UI

## 완료 기준

- 커리큘럼의 1장~11장과 부록 A~C가 문서화됐다.
- 각 장은 문제 상황, 관찰 지표, 원인 후보, 해결 선택지, 코드/구조 연결을 가진다.
- 각 개념은 정의, 실무 중요성, 흔한 오해, Visual Lab 반영 방식, Codex 누락 방지 규칙을 가진다.
- 디자인 방향과 콘텐츠 데이터 구조가 구현 전에 고정됐다.
- 다음 Phase에서 무엇을 구현하고 무엇을 금지할지 명확하다.
