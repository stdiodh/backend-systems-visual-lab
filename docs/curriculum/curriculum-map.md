# Curriculum Map

## 문서 목적

이 문서는 Backend Systems Visual Lab의 장 순서와 고정 범위를 정의한다. Codex는 이 문서에 없는 장을 임의로 추가하지 않고, 이 문서의 장을 임의로 생략하지 않는다.

## 전체 흐름

```text
01 들어가며
02 느려진 서비스, 어디부터 봐야 할까
03 성능을 좌우하는 DB 설계와 쿼리
04 외부 연동이 문제일 때 살펴봐야 할 것들
05 비동기 연동, 언제 어떻게 써야 할까
06 동시성, 데이터가 꼬이기 전에 잡아야 한다
07 IO 병목, 어떻게 해결하지
08 실무에서 꼭 필요한 보안 지식
09 최소한 알고 있어야 할 서버 지식
10 모르면 답답해지는 네트워크 기초
11 자주 쓰는 서버 구조와 설계 패턴
부록 A 처음 해보는 성능 테스트를 위한 기본 정리
부록 B NoSQL 이해하기
부록 C DB로 분산 잠금 구현하기
```

## 장별 핵심 주제

| 장 | 제목 | 핵심 주제 |
| --- | --- | --- |
| 01 | 들어가며 | 랩 목적, 사용법, 저장소 역할, Codex 작업 방식 |
| 02 | 느려진 서비스, 어디부터 봐야 할까 | 처리량, 응답 시간, p95/p99, 병목 |
| 03 | 성능을 좌우하는 DB 설계와 쿼리 | 인덱스, 실행 계획, N+1, 페이징, 트랜잭션 |
| 04 | 외부 연동이 문제일 때 살펴봐야 할 것들 | 타임아웃, 재시도, 동시 요청 제한, 서킷 브레이커, HTTP 커넥션 풀 |
| 05 | 비동기 연동, 언제 어떻게 써야 할까 | 별도 스레드, 메시징, 아웃박스, 배치, CDC |
| 06 | 동시성, 데이터가 꼬이기 전에 잡아야 한다 | 공유 데이터, race condition, 프로세스 락, DB 락, 단일 스레드 처리 |
| 07 | IO 병목, 어떻게 해결하지 | 네트워크 IO, 가상 스레드, 논블로킹 IO, 선택 기준 |
| 08 | 실무에서 꼭 필요한 보안 지식 | 인증, 인가, 암호화, HMAC, 방화벽, 감사 로그, 시큐어 코딩 |
| 09 | 최소한 알고 있어야 할 서버 지식 | OS 계정, 권한, 프로세스, 디스크, 파일 디스크립터, cron, 네트워크 명령 |
| 10 | 모르면 답답해지는 네트워크 기초 | 노드, 네트워크, 라우터, IP, 도메인, NAT, VPN, TCP/UDP/QUIC |
| 11 | 자주 쓰는 서버 구조와 설계 패턴 | MVC, 계층형, DDD, MSA, EDA, CQRS |
| A | 성능 테스트 | 테스트 종류, 포화점, 버클존, 지표, 도구, 주의 사항 |
| B | NoSQL | NoSQL 종류, 도입 기준, CAP 정리 |
| C | DB로 분산 잠금 구현하기 | 잠금 테이블, acquire/release, timeout, owner, expires_at |

## 장별 공통 구성

모든 장은 아래 구성을 가진다.

1. 문제 상황
2. 관찰 지표
3. 원인 후보
4. 핵심 개념
5. 해결 선택지
6. 코드 또는 구조 연결
7. 누락 방지 체크리스트
