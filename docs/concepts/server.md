# Server Concepts

### OS 계정과 권한

분류: 서버 운영

정의: 서버에서 프로세스와 파일 접근을 어떤 사용자와 권한으로 수행할지 정하는 기본 제어다.

실무에서 중요한 이유: 권한이 과도하면 사고 범위가 커지고, 부족하면 배포와 실행이 실패한다.

흔한 오해: root로 실행하면 운영이 편하니 좋은 방식이라고 생각한다.

Visual Lab 반영 방식: User, Group, File Permission 카드로 보여준다.

Codex 누락 방지 규칙: 서버 장에는 최소 권한 원칙과 실행 사용자 확인 명령을 포함한다.

### 프로세스

분류: 서버 운영

정의: OS에서 실행 중인 프로그램의 인스턴스다.

실무에서 중요한 이유: 서버 애플리케이션이 실제로 떠 있는지, CPU/메모리를 얼마나 쓰는지 확인해야 한다.

흔한 오해: 배포 명령이 성공하면 서비스 프로세스도 정상이라고 생각한다.

Visual Lab 반영 방식: Process/Disk/Network Tabs에서 `ps`, `top`, `systemctl` 예시를 둔다.

Codex 누락 방지 규칙: 프로세스 확인은 로그 확인과 연결한다.

### 파일 디스크립터

분류: 서버 운영

정의: OS가 파일, 소켓 같은 자원을 식별하기 위해 사용하는 핸들이다.

실무에서 중요한 이유: 제한이 낮으면 많은 연결이나 파일 사용 상황에서 장애가 발생할 수 있다.

흔한 오해: 네트워크 연결도 파일 디스크립터를 사용할 수 있다는 점을 놓친다.

Visual Lab 반영 방식: Server Resource Panel에 open files, sockets, limit를 표시한다.

Codex 누락 방지 규칙: `ulimit`, `lsof` 같은 확인 명령 카드를 포함한다.

### Cron

분류: 서버 운영 / 스케줄링

정의: 정해진 시간이나 주기로 명령을 실행하는 Unix 계열 스케줄러다.

실무에서 중요한 이유: 배치, 정리 작업, 주기적 알림 같은 운영 작업에 자주 쓰인다.

흔한 오해: 애플리케이션 스케줄러와 cron의 실행 환경 차이를 무시한다.

Visual Lab 반영 방식: Cron Schedule Card와 실행 로그 확인 흐름을 둔다.

Codex 누락 방지 규칙: 시간대, 실행 사용자, 환경변수, 로그 위치를 함께 설명한다.

### 디스크 용량

분류: 서버 운영

정의: 서버 파일 시스템에서 로그, 임시 파일, 업로드 파일, DB 파일 등이 사용할 수 있는 저장 공간이다.

실무에서 중요한 이유: 디스크가 가득 차면 로그 기록, 파일 업로드, DB 쓰기, 배포 작업이 실패할 수 있다.

흔한 오해: 애플리케이션 메모리와 CPU만 보면 서버 상태를 충분히 알 수 있다고 생각한다.

Visual Lab 반영 방식: Disk Usage Card에서 `df`, `du`, log directory growth를 연결한다.

Codex 누락 방지 규칙: 서버 장에는 디스크 full이 애플리케이션 장애로 이어지는 예시를 포함한다.

### 시간 동기화

분류: 서버 운영 / 시간

정의: 서버 시간이 기준 시간과 맞도록 NTP 같은 방식으로 동기화하는 것이다.

실무에서 중요한 이유: 인증 토큰, 로그 추적, 배치 실행, 분산 잠금 만료 시간이 서버 시간에 의존한다.

흔한 오해: 몇 분 정도의 시간 차이는 운영에 큰 문제가 아니라고 생각한다.

Visual Lab 반영 방식: Timeline Panel에서 서버 A/B의 시간 차이가 로그 순서와 expires_at 판단을 흔드는 예시를 보여준다.

Codex 누락 방지 규칙: 서버 시간 설명은 보안 토큰, cron, 분산 잠금과 연결한다.
